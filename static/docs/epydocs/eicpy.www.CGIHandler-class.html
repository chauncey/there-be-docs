Class CGIHandler

source code

Known Subclasses:
  • endorsement.erequest.EndRequestPreq

Generic Class to build/handle a webpage. CGIHandler offers convenient ways to:

  • Load macros to be used in a page template
  • Validate incoming form data
  • return text or HTML to the browser (instead of using a page template)
  • redirect a browser to a different web page
  • show a message page before redirecting the browser to a different web page

Initialization for a CGIHandler. It:

  • starts the page-load timer
  • populates default values
  • adds "std.css" to the list of stylesheets
  • adds "eiLib.js" to the list of javascript files
Parameters:
  • title - The title to send to the page template
  • pagetemplate - the relative path to the page template
    • OR - a string representing a page template
  • styles - a list of relative paths to CSS files to include in the page
  • scripts - a list of relative paths to Javascript files to include in the page
  • usemacros - provides the ability to not autoload macros for a page template
  • state - Override the agent's state. The state is detected from the agent number. If you specify a state, no agent-state lookup will happen.
  • showmenu - Set to false to not include the navmenu in your page header.
  • agent - Passed agent number trumps all others

checkData(self, vars, redirectOnFail=True)

source code 

Method to validate CGI variables against regexps.

Parameters:
  • vars - A dictionary where the key is the name of the variable and the value is a tuple describing where the variable is and the regexp to check it against. An example would look list this:

    {"REMOTE_USER":('e',r'^\d{4}$')}

    A description of the values of the tuple is as follows:

    1. either "f" or "e" for cgi.FieldStorage or os.environ respectively.
      • optionally, add an "o" to make the field optional (i.e. If the field is there, it must pass the regexp test; if not, no one cares.)
    2. regular expression to check against.
    3. (optional) function to call on failure that takes the key and data as a tuple

      (ex. ("key","f"))

  • redirectOnFail - In the case of a regexp check failure or a missing non-optional field, if set to True (the default) a friendly error page is returned and the program exits immediately, otherwise a ValueError is raised.
Raises:
  • ValueError - If a regexp check fails or a non-optional field is missing and the redirectOnFail parameter is set to False, a ValueError exception is raised.

redirect(self, link, usejs=False, delayed=False, priority=10)

source code 

Redirect the browser to a different page via an HTTP redirect or a javascript "Loading..." webpage.

Parameters:
  • link - The url of where to redirect the user to.
  • usejs - Redirect the browser with a javascript page instead of an HTTP redirect. This has the advantage of separating the URL of the redirecting page from the destination of the new page, in essence, restoring the functionality of the browser back button.
  • delayed - Set to True if you could set a higher priority destination. Redirect will be triggered by calling show or redirect or msgredir with the delayed parameter set to False.
  • priority - When using the delayed feature, having a higher priority means one message will override another. A higher priority is defined as having a lower number. For example: a message with priority 1 would override a message with priority 2.

Note: Using redirect without usejs can break some browsers' back button functionality. It is preferred to use usejs also because it displays a "Loading..." message as the next page loads.

show(self, vars, macros={})

source code 

Load the template and macros and render the page.

When calling show() to show a PageTemplate, CGIHandler provides 3 dictionaries to the pagetemplate. They are:

  1. macros contains all of the macros (*.mac or *.xml) that exist in the fragments folder.
  2. page contains various standard page information tidbits. page is generated by CGIHandler and cannot be manipulated directly. Page contains:
    • title - Page title as specified in the CGIHandler constructor
    • location - The state
    • styles - a list of stylesheets to be included in the page
    • scripts - a list of javascript files ot be included in the page
    • menu - showmenu - a Boolean for if the navigation menu should be shown
    • loadtime - time in milliseconds it took for the page to load.
    • pagecss - CSS content from self.pagecss
    • pagejs - JS content from self.pagejs
    • condcomments - conditional comments from self.condcomments
  3. vars is populated by subclasses of CGIHandler and passed in. CGIHandler does not add anything to vars.
Parameters:
  • vars - a dictionary of variables to send to the page template
  • macros - a dictionary of custom macros to use in the page template (it is rare that we would use this)

Instance Variable Details [hide private]

pagejs

self.pagejs contains straight javascript to be included in the <head> section of a page. This mechanism is primarily intended declaring page data in javascript.

pagecss

self.pagecss contains css to be included in the <head> section of a page. This mechanism is intended for pages that need very little custom CSS in order to avoid the overhead of another HTTP request. Keep in mind that large files should be linked so that the client's browser will cache them.

contenttype

self.contenttype is the Content-type: header served to the browser whenever self.show() is called. It defaults to 'text/html'.
