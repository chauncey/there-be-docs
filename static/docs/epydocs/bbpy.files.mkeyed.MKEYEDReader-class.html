readAll(self, key='', keynum=None, field=0, numerics=False, stripzeros=False)

source code 

ReadAll returns a list of all records with partial or full matches to the supplied key. Passing no key or a key=="" will RETURN ALL RECORDS in the file. This can be slow, consider yourself warned.

Parameters:
  • key - The key of the record you are searching for. If this is left blank (None) then key will be read from self._currentkey
  • keynum - Keynum refers to which key in an MKEYED file the key should be searched on.
  • field - field specifies the field of the record to return, this is for numerics
  • numerics - Specifies if numerics should be returned with the string record. If True, then the field parameter is ignored.
Returns:
A list of records that start with the key passed or if field is specified, then that specific field.
Raises:
  • BBPyKeyNotFoundError - BBPyKeyNotFoundError if key is not found

find(self, key=None, keynum=None, field=0)

source code 

find() acts just like read() with one exception. Current key pointer is only updated if a full key is matched. Unlike read() which will change it regardless of a successful key find

Parameters:
  • key - The key of the record you are searching for. If this is left blank (None) then key will be read from self._currentkey
  • keynum - Keynum refers to which key in an MKEYED file the key should be searched on.
  • field - field specifies the field of the record to return, this is for numerics
Returns:
The first field for the record that matches the key passed or if field is specified, then that specific field.
Raises:
  • BBPyKeyNotFoundError - BBPyKeyNotFoundError if key is not found
  • BBPyPartialKeyFoundException - BBPyPartialKeyFoundException if a partial key is found

Note: find() only modifies the current key pointer on a successful key match.

_walkMKEYEDTree(self, searchkey, branch)

source code 

Recursively walk the tree in search of a key. This is the heart of the whole process.

Parameters:
  • searchkey - The key of the record we are looking for.
  • branch - The branch of the tree we are searching. This is our recursive parameter
Returns:
a tuple of the index key found, its address and the index key of the next record in sequential order or None if the next record is the last record.

Note: if the key being searched for is not in the file, the tuple returned will contain the key and address of the record immediately after where the sought after key should be.

Instance Variable Details [hide private]

_keysize

The size in characters of a record key, this is not always accurate. Use self.getKeylength() to always get the accurate keysize.

_nextaddr

The offset of where the next record will go. (Not currently used).

_recordcount

The total number of records in an MKEYED file In the case of an MKEYED file, the normal file header has a value of "0". The proper value is retrieved from the MKEYED header.

keynum

In "Multi-KEYED" files, you can have multiple keys (even though we typically only use one so this is typically 0). keynum specifies which key we are referring to.

keys

A list (typically holding only 1) of the keyfield descriptions for a record (MKEYEDKeyFieldDefinition). In SQL terms, each item in this list is a member of a primary key. But since we are using BBx records, it contains each definition uses the offset and length of the key field data.
